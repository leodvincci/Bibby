## Part 5: Pricing Strategy

### Pricing Model

**Chosen model:** Freemium with subscription tiers

**Why freemium?**
- Low barrier to entry (compete with free Goodreads)
- Viral loop (free users share lists, bring new users)
- Upsell path (power users hit limits, upgrade to paid)

### Tier Structure

**Free Tier: "Reader"**
- Features: Catalog up to 100 books, basic search, reading lists (max 5), import from Goodreads
- Limits: 100 books, 5 lists, no analytics
- Goal: Acquisition and viral growth

**Paid Tier 1: "Bibliophile" - $8/month or $80/year**
- Features: Unlimited books, unlimited lists, reading analytics, tags and custom fields, API access
- Limits: Single user
- Goal: Primary revenue driver

**Paid Tier 2: "Book Club" - $20/month or $200/year**
- Features: Everything in Bibliophile + shared libraries, book club tools, group discussions
- Limits: Up to 10 members
- Goal: Team/group monetization

**Enterprise Tier: "Library Edition" - Custom pricing**
- Features: SSO, SCIM, MARC records, circulation tracking, admin controls
- Pricing: $500-2,000/month based on collection size
- Goal: Upmarket expansion

**Code Example: Pricing Tier Service**

```java
import org.springframework.stereotype.Service;
import java.util.*;

@Service
public class PricingTierService {

    public enum Tier {
        FREE_READER,
        BIBLIOPHILE_MONTHLY,
        BIBLIOPHILE_ANNUAL,
        BOOK_CLUB_MONTHLY,
        BOOK_CLUB_ANNUAL,
        ENTERPRISE_CUSTOM
    }

    public record TierLimits(
        int maxBooks,
        int maxLists,
        int maxMembers,
        boolean hasAnalytics,
        boolean hasAPI,
        boolean hasSSO,
        boolean hasCustomFields
    ) {}

    public TierLimits getLimitsForTier(Tier tier) {
        return switch (tier) {
            case FREE_READER -> new TierLimits(
                100,    // maxBooks
                5,      // maxLists
                1,      // maxMembers
                false,  // hasAnalytics
                false,  // hasAPI
                false,  // hasSSO
                false   // hasCustomFields
            );
            case BIBLIOPHILE_MONTHLY, BIBLIOPHILE_ANNUAL -> new TierLimits(
                Integer.MAX_VALUE,  // Unlimited books
                Integer.MAX_VALUE,  // Unlimited lists
                1,                  // Single user
                true,               // Analytics
                true,               // API
                false,              // No SSO
                true                // Custom fields
            );
            case BOOK_CLUB_MONTHLY, BOOK_CLUB_ANNUAL -> new TierLimits(
                Integer.MAX_VALUE,
                Integer.MAX_VALUE,
                10,                 // Up to 10 members
                true,
                true,
                false,
                true
            );
            case ENTERPRISE_CUSTOM -> new TierLimits(
                Integer.MAX_VALUE,
                Integer.MAX_VALUE,
                Integer.MAX_VALUE,  // Unlimited members
                true,
                true,
                true,               // SSO
                true
            );
        };
    }

    public record PricingDetails(
        Tier tier,
        double monthlyPrice,
        double annualPrice,
        double annualDiscount
    ) {}

    public PricingDetails getPricing(Tier tier) {
        return switch (tier) {
            case FREE_READER -> new PricingDetails(tier, 0, 0, 0);
            case BIBLIOPHILE_MONTHLY -> new PricingDetails(tier, 8, 0, 0);
            case BIBLIOPHILE_ANNUAL -> new PricingDetails(tier, 0, 80, 0.17); // 17% discount
            case BOOK_CLUB_MONTHLY -> new PricingDetails(tier, 20, 0, 0);
            case BOOK_CLUB_ANNUAL -> new PricingDetails(tier, 0, 200, 0.17);
            case ENTERPRISE_CUSTOM -> new PricingDetails(tier, 500, 6000, 0.17);
        };
    }

    /**
     * Calculate LTV based on tier and retention assumptions
     */
    public record LTVCalculation(
        Tier tier,
        double monthlyRevenue,
        double annualRetention,
        double avgLifetimeMonths,
        double ltv
    ) {}

    public LTVCalculation calculateLTV(Tier tier) {
        PricingDetails pricing = getPricing(tier);
        double monthlyRev = pricing.monthlyPrice() > 0 ?
            pricing.monthlyPrice() : pricing.annualPrice() / 12.0;

        // Retention assumptions
        double retention = switch (tier) {
            case FREE_READER -> 0.50;  // 50% annual retention (high churn)
            case BIBLIOPHILE_MONTHLY -> 0.75;
            case BIBLIOPHILE_ANNUAL -> 0.85;
            case BOOK_CLUB_MONTHLY -> 0.80;
            case BOOK_CLUB_ANNUAL -> 0.90;
            case ENTERPRISE_CUSTOM -> 0.95;
        };

        // Average lifetime in months = 1 / churn rate
        double churnRate = 1 - retention;
        double avgLifetimeMonths = churnRate > 0 ? 12 / churnRate : 120; // Cap at 10 years

        double ltv = monthlyRev * avgLifetimeMonths;

        return new LTVCalculation(tier, monthlyRev, retention, avgLifetimeMonths, ltv);
    }

    /**
     * Upgrade recommendation engine
     */
    public record UpgradeRecommendation(
        String userId,
        Tier currentTier,
        Tier recommendedTier,
        String reason,
        double expectedRevenue
    ) {}

    public Optional<UpgradeRecommendation> shouldRecommendUpgrade(
        String userId,
        Tier currentTier,
        int currentBooks,
        int currentLists
    ) {
        TierLimits limits = getLimitsForTier(currentTier);

        // Hit book limit?
        if (currentBooks >= limits.maxBooks() * 0.9 && currentTier == Tier.FREE_READER) {
            return Optional.of(new UpgradeRecommendation(
                userId,
                currentTier,
                Tier.BIBLIOPHILE_ANNUAL,
                "You're at 90% of your book limit. Upgrade for unlimited books!",
                80.0
            ));
        }

        // Hit list limit?
        if (currentLists >= limits.maxLists() * 0.8 && currentTier == Tier.FREE_READER) {
            return Optional.of(new UpgradeRecommendation(
                userId,
                currentTier,
                Tier.BIBLIOPHILE_ANNUAL,
                "You're using 4 of 5 lists. Upgrade for unlimited lists and analytics!",
                80.0
            ));
        }

        return Optional.empty();
    }
}
```

**Month 1-3: Validate willingness to pay**
- Run pre-launch survey: "Would you pay $X/month for this?"
- Test 3 price points: $5, $8, $12
- Expected outcome: Identify price elasticity

**Month 4-6: Test freemium limits**
- Experiment: Does 50 books vs 100 books free tier impact conversion?
- Hypothesis: Lower limit = higher conversion, but worse virality
- Success metric: Maximize (conversion rate × viral coefficient)

**Month 7-9: Annual vs monthly pricing**
- Experiment: Vary annual discount (10%, 17%, 25%)
- Hypothesis: 17% discount maximizes annual adoption
- Success metric: % of customers choosing annual

**Month 10-12: Add-on monetization**
- Experiment: Offer premium themes ($2/mo) and integrations ($1/mo each)
- Hypothesis: 20% of paid users buy at least one add-on
- Success metric: Revenue per user increase

### CAC and LTV by Channel

**Target unit economics:** LTV:CAC ratio > 3:1

| Channel | CAC | Conversion to Paid | LTV | LTV:CAC |
|---------|-----|-------------------|-----|---------|
| Content | $8 | 12% | $80 × 0.12 = $9.60 | 1.2:1 ❌ |
| Reddit | $0 | 15% | $80 × 0.15 = $12 | ∞ ✅ |
| BookTok | $20 | 10% | $80 × 0.10 = $8 | 0.4:1 ❌ |
| FB Ads | $40 | 8% | $80 × 0.08 = $6.40 | 0.16:1 ❌ |

**Problem:** LTV is too low! Need to improve conversion or retention.

**Solutions:**
1. Increase conversion rate to paid (target 15% vs current 8-12%)
2. Increase ARPU via upsells and add-ons
3. Improve retention (target 85% annual vs current 75%)

**Revised LTV with improvements:**
- Better onboarding → 15% conversion
- Add-ons → $100 ARPU (vs $80)
- Better retention → 90% annual (vs 75%)
- New LTV = $100 × (12 / (1-0.90)) = $100 × 120 months = $120 lifetime value

**Revised LTV:CAC ratios:**

| Channel | CAC | LTV | LTV:CAC |
|---------|-----|-----|---------|
| Content | $8 | $18 | 2.25:1 ⚠️ |
| Reddit | $0 | $18 | ∞ ✅ |
| BookTok | $20 | $15 | 0.75:1 ❌ |
| FB Ads | $40 | $12 | 0.3:1 ❌ |

**Still not great!** This shows we need even higher conversion or retention, or must keep CAC very low.

**Realistic approach:**
- Focus on $0-10 CAC channels only (content, Reddit, partnerships)
- Don't use paid ads until LTV improves to $50+
- Build viral loops to reduce dependence on paid acquisition

```java
import org.springframework.stereotype.Service;
import java.time.LocalDate;
import java.util.*;

@Service
public class GrowthLoopService {

    public enum LoopType {
        CONTENT_SHARING,
        REFERRAL_PROGRAM,
        INTEGRATION_NETWORK
    }

    public record LoopMetrics(
        LoopType loopType,
        LocalDate period,
        int totalUsers,
        int loopParticipants,
        double participationRate,
        int invitesSent,
        double invitesPerParticipant,
        int conversions,
        double conversionRate,
        double kFactor
    ) {}

    /**
     * Calculate k-factor for a growth loop
     */
    public LoopMetrics calculateLoopMetrics(
        LoopType loopType,
        LocalDate period,
        int totalUsers,
        int participants,
        int invitesSent,
        int conversions
    ) {
        double participationRate = totalUsers > 0 ?
            participants / (double) totalUsers : 0;

        double invitesPerParticipant = participants > 0 ?
            invitesSent / (double) participants : 0;

        double conversionRate = invitesSent > 0 ?
            conversions / (double) invitesSent : 0;

        double kFactor = participationRate * invitesPerParticipant * conversionRate;

        return new LoopMetrics(
            loopType,
            period,
            totalUsers,
            participants,
            participationRate,
            invitesSent,
            invitesPerParticipant,
            conversions,
            conversionRate,
            kFactor
        );
    }

    /**
     * Identify optimization opportunities for growth loops
     */
    public record LoopOptimization(
        LoopType loopType,
        String bottleneck,
        double currentValue,
        double targetValue,
        double potentialImpact,
        String recommendation
    ) {}

    public List<LoopOptimization> identifyOptimizations(LoopMetrics metrics) {
        List<LoopOptimization> optimizations = new ArrayList<>();

        // Low participation rate?
        if (metrics.participationRate() < 0.25) {
            double potentialImpact = 0.30 * metrics.invitesPerParticipant() * metrics.conversionRate();
            optimizations.add(new LoopOptimization(
                metrics.loopType(),
                "Low participation rate",
                metrics.participationRate(),
                0.30,
                potentialImpact - metrics.kFactor(),
                "Add in-app prompts and email campaigns to encourage sharing"
            ));
        }

        // Low invites per participant?
        if (metrics.invitesPerParticipant() < 2.0) {
            double potentialImpact = metrics.participationRate() * 3.0 * metrics.conversionRate();
            optimizations.add(new LoopOptimization(
                metrics.loopType(),
                "Low invites per participant",
                metrics.invitesPerParticipant(),
                3.0,
                potentialImpact - metrics.kFactor(),
                "Make sharing easier with one-click social sharing and bulk invites"
            ));
        }

        // Low conversion rate?
        if (metrics.conversionRate() < 0.20) {
            double potentialImpact = metrics.participationRate() * metrics.invitesPerParticipant() * 0.25;
            optimizations.add(new LoopOptimization(
                metrics.loopType(),
                "Low conversion rate",
                metrics.conversionRate(),
                0.25,
                potentialImpact - metrics.kFactor(),
                "Improve landing page UX and add social proof (testimonials, user count)"
            ));
        }

        return optimizations.stream()
            .sorted(Comparator.comparingDouble(LoopOptimization::potentialImpact).reversed())
            .toList();
    }
}
```

```java
import org.springframework.stereotype.Service;
import java.time.LocalDate;
import java.time.temporal.ChronoUnit;
import java.util.*;

@Service
public class RetentionAnalysisService {

    public record CohortRetention(
        LocalDate cohortMonth,
        int cohortSize,
        Map<Integer, Double> retentionByMonth  // Month 0, 1, 2, 3...
    ) {}

    /**
     * Calculate cohort retention rates
     */
    public CohortRetention calculateCohortRetention(
        LocalDate cohortMonth,
        List<UserActivity> cohortUsers
    ) {
        int cohortSize = cohortUsers.size();
        Map<Integer, Double> retention = new HashMap<>();

        for (int month = 0; month <= 12; month++) {
            LocalDate targetMonth = cohortMonth.plusMonths(month);
            long activeUsers = cohortUsers.stream()
                .filter(u -> wasActiveInMonth(u, targetMonth))
                .count();

            double retentionRate = activeUsers / (double) cohortSize;
            retention.put(month, retentionRate);
        }

        return new CohortRetention(cohortMonth, cohortSize, retention);
    }

    private boolean wasActiveInMonth(UserActivity user, LocalDate month) {
        // Check if user had any activity in the target month
        return user.activityDates().stream()
            .anyMatch(date ->
                date.getYear() == month.getYear() &&
                date.getMonth() == month.getMonth()
            );
    }

    /**
     * Calculate "smile graph" - retention curve
     */
    public record RetentionCurve(
        List<Integer> months,
        List<Double> retentionRates,
        String curveType  // "smile", "declining", "flat"
    ) {}

    public RetentionCurve analyzeRetentionCurve(CohortRetention cohort) {
        List<Integer> months = new ArrayList<>();
        List<Double> rates = new ArrayList<>();

        for (int m = 0; m <= 12; m++) {
            months.add(m);
            rates.add(cohort.retentionByMonth().getOrDefault(m, 0.0));
        }

        // Determine curve type
        double month1 = cohort.retentionByMonth().getOrDefault(1, 0.0);
        double month6 = cohort.retentionByMonth().getOrDefault(6, 0.0);
        double month12 = cohort.retentionByMonth().getOrDefault(12, 0.0);

        String curveType;
        if (month12 > month6) {
            curveType = "smile";  // Retention improves over time (great!)
        } else if (month12 > month1 * 0.5) {
            curveType = "flat";   // Stable retention (good)
        } else {
            curveType = "declining";  // Concerning churn (bad)
        }

        return new RetentionCurve(months, rates, curveType);
    }

    public record UserActivity(
        String userId,
        LocalDate signupDate,
        List<LocalDate> activityDates
    ) {}
}
```

### Growth Accounting Framework

**Track 3 components of growth:**

1. **New users:** Signups from acquisition channels
2. **Resurrected users:** Previously churned users who return
3. **Churned users:** Active users who stop using product

**Growth formula:**
```
Net user growth = New + Resurrected - Churned
```

**Example Month:**
- New: 500
- Resurrected: 50
- Churned: 100
- Net growth: 500 + 50 - 100 = +450 users

**Healthy growth indicators:**
- Churned < 10% of active base monthly
- Resurrected > 5% of churned (successful win-back campaigns)
- New growing month-over-month

```
  ↓
Iteration
```

Most people start with tactics. Winners start with vision.

### Common GTM Mistakes

❌ **Launch without ICP clarity:** "Everyone" is not a target market
❌ **Compete on price:** Race to zero is a race to death
❌ **Ignore unit economics:** Revenue without profit is vanity
❌ **Optimize for vanity metrics:** Signups don't matter if they churn
❌ **Copy competitors:** Differentiation comes from being different, not better
❌ **Scale too early:** Pouring fuel on a fire that isn't burning yet

### What Makes a Great GTM

✅ **Clear ICP:** You can describe your ideal customer in detail
✅ **Differentiated positioning:** You own a unique space in the market
✅ **Proven value:** Customers say "I can't live without this"
✅ **Efficient acquisition:** LTV:CAC > 3:1
✅ **Strong retention:** Annual churn < 10%
✅ **Viral growth:** k-factor > 0.3
✅ **Expanding revenue:** Net revenue retention > 100%

### Your GTM is Never Done

Markets shift. Competitors emerge. Customer needs evolve.

Your GTM playbook is a living document. Revisit it quarterly. Update assumptions. Kill what's not working. Double down on what is.

The companies that win aren't those with the best initial GTM plan. They're the ones that iterate fastest.
